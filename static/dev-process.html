コード実装。FastAPI + Playwright によるバックエンド、Tailwind CSS によるフロントエンドを一気通貫で生成

ツール Render

クラウドホスティング。GitHub連携で自動デプロイ。Dockerコンテナで Chromium + Python を実行

DAY 5 — 4月12日（土）

SUUMO完全復活・全3サイト稼働

👤 てるさんのプロンプト

「L-008の作業を再開。引き継ぎドキュメントを読んで、Render環境でSUUMO/HOME'Sが0件になる問題を調査して」

🔍 根本原因の特定（3層の問題）
原因① HOME'S — ページロードのタイムアウト

wait_until="domcontentloaded" だと874KBの重いページがRenderの低CPUでは30秒以内に完了しない → wait_until="load" に変更して解決

原因② SUUMO — DOM要素の非表示ロード

要素はDOMに存在（query_selector_allで20件確認）するが、Playwrightのwait_for_selectorはstate='visible'を要求してタイムアウト。SUUMOはJSで後から表示する動作 → ポーリング方式に変更

原因③ SUUMO — データセンターIPの間欠ブロック

RenderのIPからのアクセスに対し503を間欠的に返す → バックオフ付き最大3回リトライで対処

主要な設計判断

3サイト構成（HOME'S + SUUMO + ふれんず）

アットホームはWAF/CAPTCHA、楽待はCloudflare bot対策で断念。安定的にデータ取得できる3サイトに集中

プラグイン構造（サイト追加が容易）

SCRAPER_REGISTRY に登録するだけで新サイトを追加可能。各スクレイパーは BaseScraper を継承

100万円価格ガード（賃貸除外）

models.py の1箇所で全サイト一律適用。月額5万円の賃貸物件を確実にフィルタ

順次実行（メモリ節約）

Render Free の 512MB RAM では Chromium 3つ同時起動は不可。1サイトずつ順次実行

単一HTML構成

ビルドツール不要。Tailwind CDN + vanilla JS でフロントエンドを完結
